# AutoFix - Reparación de Vehículos a Domicilio

## Descripción del Proyecto

Este proyecto consiste en el desarrollo de una página web básica para **AutoFix**, un negocio de reparación de vehículos a domicilio. El objetivo principal es proporcionar una solución digital para que los clientes puedan solicitar servicios, consultar precios, agendar citas y realizar pagos en línea (simulados).

El diseño de la página se basa en el wireframe proporcionado, enfocándose en la **funcionalidad** y la **responsividad** utilizando únicamente HTML y CSS.

## Requisitos Implementados

La página web cumple con los siguientes requisitos funcionales:

| Componente | Descripción |
| :--- | :--- |
| **Servicios y Precios** | Sección visual que muestra los servicios principales (Cambio de aceite, Revisión de frenos, Diagnóstico eléctrico) con sus respectivos precios en MXN. |
| **Formulario de Solicitud** | Permite al usuario ingresar su nombre, seleccionar el tipo de reparación y añadir una descripción. |
| **Sistema de Citas** | Un campo de entrada (`datetime-local`) para que el cliente pueda programar la fecha y hora de la reparación. |
| **Pago en Línea (Simulado)** | Una sección para ingresar información de pago (número de tarjeta, nombre del titular, CVV) con fines de simulación. |

## Tecnologías Utilizadas

*   **HTML5**: Para la estructura y el contenido de la página (`index.html`).
*   **CSS3**: Para el diseño, la presentación y la responsividad (`style.css`).

## Gestión de Código Fuente (Git)

El proyecto utiliza **Git** para el control de versiones, asegurando un flujo de trabajo colaborativo y la trazabilidad de los cambios.

### Flujo de Trabajo

Se ha inicializado un repositorio Git en el directorio del proyecto, con un *commit* inicial que incluye la estructura base de HTML y los estilos CSS.

*   **Control de Versiones**: Se utiliza Git para registrar cada cambio significativo.
*   **Commits**: El primer commit (`Initial commit: AutoFix website structure (HTML and CSS)`) establece la base del proyecto.
*   **Colaboración**: El repositorio está listo para ser subido a una plataforma como GitHub, donde el equipo puede usar ramas, *pull requests* y *tags* para un desarrollo eficiente.
